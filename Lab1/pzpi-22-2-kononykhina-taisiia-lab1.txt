1 BUSINESS REQUIREMENTS (БІЗНЕС-ВИМОГИ)

1.1 Background (Передумови)


     У сучасному світі багато людей прагнуть займатися активним відпочинком та хобі, такими як туризм, кемпінг, гірськолижний спорт та інші види розваг. Проте придбання дорогого спорядження для таких заходів не завжди є виправданим, особливо якщо користування ним є нерегулярним або сезонним. Також не кожен має можливість зберігати велике спорядження вдома через обмежений простір.
     Програмна система для оренди спорядження для хобі та розваг пропонує зручне рішення цієї проблеми. Вона дозволяє користувачам легко знаходити, бронювати та орендувати необхідне спорядження через інтуїтивно зрозумілий інтерфейс, забезпечуючи доступ до різних видів спорядження без потреби в його постійному зберіганні або купівлі. Система автоматизує процеси бронювання, оплати та відгуків, що зменшує час і зусилля, необхідні для пошуку та організації оренди.


1.2 (Business Opprtunity) Бізнес можливості


     У рамках виконання роботи було визначено ключові вимоги до програмної системи оренди спорядження для хобі та розваг. Аналіз існуючих рішень у сфері оренди спорядження, таких як Mandrivnyk (https://prokat.mandrivnyk.kiev.ua/ua/) та FreeRide (https://freeride.com.ua), дозволив оцінити їхні можливості та виявити обмеження у функціональності й користувацькому досвіді.
     Mandrivnyk — це онлайн-платформа для оренди спорядження для хобі та розваг, що дозволяє користувачам знаходити та орендувати необхідне спорядження для активного відпочинку. Платформа пропонує різноманітні види

спорядження, включаючи туристичне, кемпінгове та гірськолижне обладнання. До переваг Mandrivnyk можна віднести широкий вибір спорядження, зручний інтерфейс для бронювання та можливість перевірки наявності спорядження перед орендою. Однак платформа обмежена доступністю спорядження лише у певних регіонах, а процес оплати не завжди є гнучким.
     FreeRide — це сервіс оренди спорядження для активного відпочинку, зокрема спорядження для гірськолижного спорту та сноубордингу. Платформа дозволяє користувачам орендувати спорядження на різний термін, що зручно для короткострокових поїздок. Основні переваги FreeRide включають простоту процесу оренди, можливість швидкої онлайн-оплати та доступ до різноманітних типів спорядження. Проте, обмежений асортимент спорядження та відсутність інтеграції з мобільними додатками можуть знижувати зручність використання платформи для деяких користувачів.


1.3 Способи монетизації


● Плата за оренду: основне джерело доходу — комісія з кожної оренди спорядження. Користувачі платять за послугу оренди спорядження, а платформа отримує частку від цієї транзакції.
● Додаткові послуги: можливість надання додаткових послуг, таких як страхування спорядження, доставка обладнання до місця призначення, консультації щодо вибору відповідного спорядження. Ці послуги можуть надаватися за додаткову плату, що збільшує загальний дохід.

2 VISION OF THE SOLUTION (КОНЦЕПЦІЯ РІШЕННЯ)

2.1 Vision Statement (Окреслення концепції)


     Система надаватиме зручний та швидкий доступ до широкого асортименту спорядження для хобі та розваг, щоб задовольнити потреби користувачів у різних активностях — від походів та кемпінгу до зимових видів спорту та водних розваг. Платформа стане посередником між орендодавцями та орендарями, забезпечуючи прозорі умови оренди, легкий процес бронювання, безпечну оплату та можливість доставки спорядження.


2.2 Major Features (Головна функціональність)


     Функції, що будуть виконуватися системою оренди спорядження для хобі та розваг:
     MF 1) Пошук спорядження, фільтрація результатів.
МF 2) Перегляд/додавання товарів до кошика користувачем.
MF 3) Оренда спорядження.
MF 4) Реєстрація користувача.
MF 5) Система аналітики.

    2.3      Backend Features (Серверна частина)
     
     Основні можливості, реалізовані на серверній стороні системи оренди спорядження:
BF 1) Додавання, редагування, перегляд та видалення товарів (для адміністраторів/власників спорядження).
BF 2) Реєстрація користувачів, авторизація та логін через систему аутентифікації.
BF 3) Можливість залишати відгуки на товари, а також їх видалення.
BF 4) Додавання товарів до кошика для формування замовлення.
BF 5) Збереження товарів у вішліст (обрані) для швидкого доступу пізніше.
BF 6) Створення та видалення замовлень користувачами.

2.4      Mobile Features (Мобільна частина)

     Мобільна версія системи забезпечує базову взаємодію користувача з платформою:
MF 1) Перегляд наявного спорядження, включаючи детальну інформацію про кожен товар.
MF 2) Авторизація та вхід до облікового запису.
MF 3) Додавання спорядження до кошика для подальшого оформлення замовлення.
MF 4) Додавання товарів до списку обраного для збереження улюблених позицій.



3 SCOPE AND LIMITATIONS (РАМКИ ТА ОБМЕЖЕННЯ ПРОЄКТУ)


3.1 Scope of Initial Release (Рамки первинного випуску)


     Перелік функціональності для програмної системи оренди спорядження для хобі та розваг.
     MF 1-1) Зберігання інформації про доступне спорядження, його тип, вартість оренди.
     MF 1-2) Інтеграція з платіжними системами для безпечної та швидкої оплати оренди.
     MF 1-3) Авторизація та реєстрація користувачів.
     MF 1-6) Надання можливості орендодавцям додавати нове спорядження та керувати його доступністю.

3.2 Scope of Subsequent Releases (Рамки наступних випусків)


     Перелік функціональності для веб та мобільних клієнтів програмної системи оренди спорядження для хобі та розваг.
     
     MF 2-2) Відправка сповіщень через email або push-повідомлення для підтвердження бронювання, завершення оренди та спеціальних пропозицій.
     MF 2-3) Аналітика.
     MF 2-4) Система оцінки та відгуків.

4 BUSINESS CONTEXT (БІЗНЕС-КОНТЕКСТ)

4.3 Operating Enviroment (Робоче середовище)


     Система буде використовуватись користувачами, які можуть знаходитися в різних регіонах та часових зонах. Основним завданням є забезпечення доступу до системи в будь-який час доби для перегляду спорядження, його оренди, керування послугами та платежами. Система передбачає інтеграцію з мобільними додатками та веб-платформою, що потребує стабільного інтернет- з’єднання, високої надійності та продуктивності.
Мови програмування:
     JavaScript (React): використовуватиметься для створення інтерактивного інтерфейсу веб-додатка, що забезпечує зручність користування.
     Node.js/Express.js: використовуватиметься для розробки серверної частини, забезпечуючи ефективну роботу REST-full API та інтеграцію з базою даних.

Інфраструктура серверної частини:
     Express: веб-фреймворк для створення серверної частини, забезпечує обробку HTTP-запитів, маршрутизацію та підключення middleware.
     Mongoose: бібліотека для роботи з MongoDB, яка надає інструменти для створення схем та управління даними.
     Morgan: логгер запитів, який допомагає відслідковувати запити до сервера під час розробки.
     Cookie-parser: Middleware для обробки кукі, що полегшує роботу з файлами cookie.
     CORS: Middleware, який дозволяє взаємодію між сервером і клієнтами з різних доменів.

     JWT: система авторизації на основі токенів для безпечного доступу користувачів до різних частин системи.
     Upload Folder Creation Script: автоматичне створення директорій для завантаження файлів (products, customize, categories) під час запуску сервера.
     Router Modules: система маршрутизації для управління авторизацією, категоріями, продуктами, платежами, замовленнями та налаштуваннями.

Інфраструктура фронтенд частини:
– React: бібліотека для створення веб-інтерфейсу.
– Material-UI: для стилізації інтерфейсу.

Інфраструктура мобільного додатку:
– Kotlin.
